Sales and customer analysis for marketing campaign

MySQL project

Full code available here

  This project mainly demonstrates cleaning/processing/wrangling the retail data. During this process, I expect to provide useful information and the insight which can be helpful to set marketing plans or business strategies. Main tool of this analysis is MySQL and the Dataset is Shopping cart Database which contains Australian reatail sales data sourced from Kaggle. The project is formed as process of 'Question -> SQL query -> Output -> Interpretation' through out the post.

NOTE: This dataset is randomly generated by the kaggle user and it was only used to demonstrate the skill. Therefore, it may not represent the real market or customer trend

Project skill goals

  • Data Manipulation
  • Data Cleaning
  • Logical Thinking
  • Relational Database System

Used tools

  • MySQL

Exploratory Data Analysis

 The dataset has total 4 csv files (orders,customers,products,sales). I have created the table and assign correct data type for each variable.

Let's have a look at all tables !

Table 1. Customers table

Customer table contains customer personal information such as name, gender, age. Customer_id is the Primary key of this table.

Table 2. Orders table

The orders table have information about each order made by customer. order_id is unique and primary key. Each order also shows when it was made and delivered.

Table 3. Products table

  Products table has the detail information of each product sold.

Table 4. Sales table

  Lastly, Sales table has information of profit by product id. sales_id is primary key and order_id can be duplicate values.

Figure 1. Database Relationship

Figure 1 represents how each table is connected to each other. Later part of this project, several tables will be combined to extract some information.
